无聊搞搞angr

Posted on 2026-01-12

忘记在哪里偶遇了angr这个东西，正好有时间就学了一下，这里稍做记录

题目来源：jakespringer/angr_ctf

这个网站有17道题，由浅入深地考察angr的使用

可以用git克隆这个仓库到本地，里面有个dist的文件夹，放了编译好的二进制文件

git clone https://github.com/jakespringer/angr_ctf.git

前人的建议是angr的安装建议在虚拟环境里面，大概就是这样

python3 -m venv angr_env

source angr_env/bin/activate

pip install angr

之后开启虚拟环境运行写好的py就行了

python3 -m venv ~/angr_env # 创建虚拟环境

source ~/angr_env/bin/activate # 激活，提示符会变成 (angr_env)

脚本如下：

简短的说就是simulation.explore(find = lambda s : b’Good Job’ in s.posix.dumps(1) , avoid = lambda s : b’Try again’ in s.posix.dumps(1))

03_angr_symbolic_registers

出题人出这道题的时候angr还没办法很好的处理多个输入的问题，而现在可以了，但是为了学习还是按照之前的方法做

假设没办法处理多个输入的scanf，那么就需要人工处理，直接从输入之后的地方开始执行程序，再用人工将输入的内容放到寄存器中，达成一个人工输入的效果

首先是自定义程序入口，用到的是这个blank_state，它可以接收一个addr参数，传入地址

根据图片不难分析出我们应该在0x8048980处开始执行，不然会错过寄存器的值存入变量中的赋值过程

然后我们需要手动定义三个符号位向量，这里用到了claripy.BVS(name,size)，size是一个整数，name是自己给定义的名字

接着将符号位向量分别喂给相应的寄存器，这里需要用ida分析

可以发现，这里分别压入了v10,v14,v18的地址，然后压入格式字符串的地址

也就是输入的三个数分别存入了v18，v14，v10的地址，而下面mov指令可以看出，输入的数字分别存入了ecx，ebx，edx，因此，将三个符号位向量分别写入这三个寄存器即可（用到了init_state.regs.??? = xxx）

剩下的步骤基本上就是常规的explore

还有一点要注意的是最后求解成功之后，三个未知数不能直接posix.dumps输出，而是需要用state.solver.eval求解，因为符号变量经过运算后得到的是符号表达式，需要使用eval将表达式转化为具体的数值

顺便补充一下f-string指定输出格式的方法：f”{表达式:格式说明符}”

print(f”hex: {num:x}”) # hex: ff
print(f”hex: {num:08x}”) # hex: 000000ff (8位宽度，前导0填充)

print(f”HEX: {num:X}”) # HEX: FF
print(f”HEX: {num:08X}”) # HEX: 000000FF
